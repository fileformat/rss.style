What is this?

Every website that updates routinely should have an RSS or Atom news feed that people (and search engines) can use to watch for changes.

By default, these files look horrible when a human looks at them in a browser. But there is a simple way to make them look better: RSS Style!

If you are not a webmaster, checkout out What is a feed? and Picking a news reader for how to get started!

How do I use it?

You need to know the name of your RSS/Atom file. It is usually something like feed.xml, atom.xml or rss.xml. You may be able to use the Feed Analyzer to help you find it. These examples use feed.xml, but you should replace it with the name of your feed file.

Add a line to your feed.xmlfile. At the top of the file, it will look like: 

<?xml version="1.0" encoding="UTF-8"?>
<rss ...>
...

Add a line in the middle so it looks like: 

<?xml version="1.0" encoding="UTF-8"?>
<rss ...>
<script src="https://www.rss.style/js/rss-style.js" xmlns="http://www.w3.org/1999/xhtml"></script>
...

If your feed is an Atom feed, it will be slightly different: <feed...> instead of <rss...> and the script will be atom-style.js

Available scripts

  • rss-style.js - for RSS feeds (minified)
  • atom-style.js - for Atom feeds (minified)

Questions and Answers

Which file do I add it to?

Your feed file is often named feed.xml but can be anything. You can use the Feed Analyzer to find the URL of your feed: put it your website's URL and it will display the feeds that it finds.

How does this work?

The <script> replaces the content with an XHTML version. The process raw XML into HTML. This has been supported by all major browsers for a long time.

Why am I still getting a wall of text? Or a download prompt?

Your web server needs to send the correct Content-Type header for your feed. You need to use either text/xml or application/xml. It does not work with application/rss+xml or application/atom+xml. You can test it with the Content-Type Testing page.

I don't want to use a script from a remote site. May I copy it to my server?

Yes! The scripts (and everything else on this site) are MIT licensed.

Known Issues

There is a flash of unstyled content

I haven't been able to figure a really good way to get rid of it. You can add a CSS stylesheet to the feed, but this only makes the flash a little shorter. Let me know if you have any ideas!

Post bodies are shown as raw HTML

This is because the posts need to be XHTML. I'm not using a full HTML parser, so if there are any errors decoding or embedding the post body as HTML, it will just be shown as raw text.
